meshTerm User Guide

meshTerm is a professional SSH terminal client built for iOS and iPadOS. It provides seamless remote server access via password or public-key authentication, featuring specialized deep integration with Tailscale for secure network management on the go.

Prerequisites

For all connections

  • A server with SSH enabled (sshd) and accessible from your device's current network.
  • Connection details: Hostname/IP, Port (default 22), and valid Username.

For SSH key authentication Pro

  • An Ed25519 SSH key pair. You can generate one within meshTerm or import an unencrypted OpenSSH key.
  • The corresponding public key must be appended to the server's ~/.ssh/authorized_keys file.

For Tailscale SSH Pro

  • Tailscale app installed: meshTerm routes through the VPN tunnel, requiring the Tailscale app to be active.
  • Device & Server must both be on the same Tailnet.
  • Server must have Tailscale SSH enabled in the ACLs.

Getting Started

Adding your first host

  1. Launch meshTerm. The host list will be empty.
  2. Tap the + icon in the top-right.
  3. Label: Provide a nickname (e.g., "Web Server").
  4. Hostname/IP: The address of your server.
  5. Port: Default is 22.
  6. Username: Your remote account name.
  7. Authentication: Choose your method and enter credentials.
  8. Tap Add to save.

Managing SSH Hosts

Your hosts are displayed on the main dashboard. You can interact with them using native iOS gestures:

  • Edit: Swipe right on a host and tap Edit, or long-press the row.
  • Delete: Swipe left on a host and tap Delete.
  • Host Limit: Free users can save 1 host. Pro removes this limit.

Connecting to a Host

When you initiate a connection, meshTerm performs the following steps:

  1. Resolves the hostname and opens a TCP connection.
  2. Verifies the server's host key against saved fingerprints.
  3. Authenticates via the method you configured.
  4. Requests a PTY (Pseudo-Terminal) and launches the shell.

Environment Variables

You can attach custom environment variables to any host. These are sent to the shell session upon connection. Edit the host and locate the Environment Variables section to add key-value pairs.

Disconnecting

To end a session, close the terminal view or type exit / logout (or use Ctrl+D).

Authentication Methods

Password

Stored securely in the iOS Keychain. They are never saved in plain text on the filesystem.

SSH Key Pro

The private key remains in the Keychain. meshTerm signs the authentication challenge locally. You must copy your Public Key to the server first.

Tailscale SSH Pro

For Tailnet-enabled servers, Tailscale handles identity at the network layer. On the remote host, ensure you have run:

sudo tailscale up --ssh
Tailscale SSH authentication is handled entirely by the Tailscale app — meshTerm does not send any credentials for these connections.

SSH Key Management Pro

Access management via Settings → SSH Keys.

Generating a New Key

meshTerm creates 32-byte Ed25519 seeds stored in the Secure Enclave (where possible) via the iOS Keychain.

Importing Keys

Must be Ed25519 and Unencrypted (no passphrase). To strip a passphrase from an existing key, use:

ssh-keygen -p -N "" -f ~/.ssh/id_ed25519

Key Storage Security

  • Private keys are marked WhenUnlockedThisDeviceOnly.
  • They are not backed up to iCloud and cannot be moved to other devices.

Tailscale Integration Pro

Getting an API Token

  1. Sign in at tailscale.com.
  2. Go to Settings → Personal Settings → Keys.
  3. Generate a Read-Only token.
  4. Paste into Settings → Tailscale → API Token in meshTerm.

Browsing your Tailnet

Tap the globe icon in the host list toolbar to see all authorized devices. You can add them as hosts instantly; they default to Tailscale SSH authentication.

Snippet Vault Pro

The Snippet Vault allows one-tap command execution. Default folders include Linux (admin tools like systemctl) and OpenClaw.

  • Adding: Use the + in the Vault to add custom titles and tags.
  • Searching: The search bar scans titles, text, and tags.

Extended Keys

Dismiss the keyboard to see the Extended Row: F1–F12, Home, End, Page Up, and the Symbol Strip (|, ~, $, /, \, etc.).

Security

Known Hosts (TOFU)

meshTerm uses Trust On First Use. You will be prompted to accept a fingerprint on the first connection. Manage these in Settings → Known Hosts.

App Lock

Protect your sessions with Face ID, Touch ID, or Passcode. Enabled in Settings → App Lock. It requires a device-level passcode to be set in iOS Settings.

Troubleshooting

Connection Errors

  • "Could not resolve": Check DNS or internet connection. Ensure .local hostnames are on the same WiFi.
  • "Connection Refused": Check if sshd is running: sudo systemctl status ssh. Ensure Port 22 is open.
  • "Hanging at Connecting": Ensure Tailscale app is connected if using a Tailnet IP.

Authentication Failures

  • No SSH Key Selected: Edit host → Authentication → Choose Key.
  • Key Rejected: Check server permissions (.ssh should be 700, authorized_keys should be 600).
  • Passphrase Errors: meshTerm does not support encrypted keys. Strip the passphrase before importing.

Security Warnings

Host Key Mismatch: If your server was rebuilt, delete the old entry in Settings → Known Hosts. Otherwise, be wary of a Man-in-the-Middle attack.
